元数据区域使用指南
=================

目的
----
元数据区域用于从“提示词”视角呈现提示词的价值维度、使用说明、风险提示及辅助信息（推荐模型、估算 tokens、预览媒体等），帮助用户快速判断、复用与分享提示词。

字段说明（简要）
---------------
- 推荐模型 (recommendedModels)：建议用于运行该提示词的模型列表（字符串数组）。
- 估算 Tokens：基于提示词正文的字符数做的粗略估算，仅用于快速判断成本和复杂度。
- 复杂度：根据 token 数分档（低/中/高），便于快速评估执行难度。
- 价值维度 / 场景：读者友好的一句话摘要，基于类别与标签自动推断，帮助搜索与聚类（只读）。
- 使用说明 (usageNotes)：面向使用者的简短步骤与小技巧（1-3 句）。示例：如何结构化输入、常用参数、变量说明。
- 注意事项 / 风险 (cautions)：列出可能导致错误或不安全输出的条件和缓解建议（1-2 句）。
- 预览媒体 URL (previewMediaUrl)：图片/视频/音频链接，支持内嵌预览（图片、video、audio）。仅用于演示与分享，请保证外链可访问。
- 来源信息 (source / sourceAuthor / sourceUrl)：可选来源与作者/链接，便于溯源与版权提示。

交互要点
--------
- 自动生成元信息：点击“自动生成元信息”会调用模型（若可用）或启发式规则填充若干字段。自动生成结果为建议，用户须确认或编辑后保存。
- 自动标记：根据提示词自动推荐标签，用户可选择接受或手动编辑。
- 媒体预览：在 `previewMediaUrl` 中填写外链后会尝试按文件后缀渲染 video / audio / image；若预览失败，请检查链接或打开新标签页查看。
- Token 实时估算：提示词或示例变更时会更新估算 token 数，供成本/复杂度评估参考。
- 导出与复制：建议后续支持“导出 metadata JSON”用于提示库导入/导出（当前实现保留表单一致性）。

最佳实践（简短）
-------------
- 在 `usageNotes` 中写明必要输入格式与变量说明，例："{name} 为用户姓名，{topic} 为主题关键词"。
- 在 `cautions` 提示潜在敏感数据风险或输出边界（如“避免在生产环境直接使用未经审查的生成内容”）。
- 使用 `recommendedModels` 指出适合的模型类别（生成文本 vs 生成图像），提高复用效果。
- 当需要演示输出时，使用 `previewMediaUrl` 指向可靠托管资源（CDN 或受控服务器）。
